Privacy Policy for song

Last updated: 2026-06-29

Overview

song is a private automation app used by its owner to collect public Threads content from selected public Threads profiles for a local financial morning report workflow. The app is not a public social network, advertising product, or user-facing publishing service.

Data We Access

When authorized through the Threads API, song may access public Threads posts, public profile usernames, post timestamps, permalinks, post text, public media metadata, and public conversation replies that are available through approved Threads API permissions. The app may also access app-scoped tester account data during development and review.

How We Use Data

Data is used only to prepare internal financial monitoring and morning report summaries. song does not sell, rent, or share Threads API data with advertisers or data brokers. The app does not use Threads API data to build public user profiles or to make automated decisions about individuals.

Storage and Retention

Collected data is stored locally on the app owner's machine as part of the MoneyMachine reporting workflow. Data is retained only as long as needed for report generation, debugging, audit, and rollback. Access tokens are stored locally and are not published in reports or public pages.

Sharing

song does not intentionally disclose collected Threads API data to third parties. Generated reports may include short excerpts or links to public Threads posts when they are relevant to the owner's private financial monitoring workflow.

User Control and Deletion

If you believe song has stored data related to your Threads account and you want it deleted from the local workflow, follow the instructions on the User Data Deletion Instructions page.

Contact

For privacy or deletion requests, contact the app owner at the contact email configured in the Meta Developer app dashboard for song.
